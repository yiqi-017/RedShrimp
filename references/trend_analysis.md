# 小红书趋势分析与爆款预测 (Trend Analysis)

## 概述
当用户“不知道发什么”“最近什么容易火”“帮我看热帖方向”时，优先通过本地 `xiaohongshu-mcp` 做实时观察，而不是只给经验判断。

目标不是告诉用户一个泛赛道，而是给出 **2-3 个马上能做的具体切入点**，并把这次观测结果追加保存到 `history/trends_data.txt`。

## 执行前提
1. 先确认 `xiaohongshu-mcp` 已连接，且存在 `check_login_status`、`search_feeds`、`get_feed_detail`。
2. 再调用 `check_login_status` 确认已经登录。
3. 若未连接或未登录：
   - 不得伪装成实时分析；
   - 优先读取 `history/trends_data.txt` 最近一次 run；
   - 若 history 也没有有效数据，再退回本文末的常青 fallback。

## 趋势侦测 SOP
### 1. 明确扫描范围
- **有明确赛道**：优先按赛道生成 2-3 个关键词，再做搜索验证。
  - 例如：
    - 美食探店 → `广州美食`、`探店`、`约会餐厅`
    - 职场 → `打工人`、`简历`、`副业`
    - 留学 → `留学生`、`申请`、`QS`
    - 穿搭 → `穿搭`、`梨形身材`、`通勤穿搭`
- **没有明确赛道**：使用泛词扫描，例如 `打工人`、`学生党`、`搞钱`、`减脂`、`探店`。

### 2. 先看 Feed，再看 Search
1. 优先尝试 `list_feeds` 观察推荐流，样本 10-20 条即可。
2. 统计高频出现的：
   - 赛道/人群
   - 情绪词
   - 场景词
   - 地域词
3. 如果 `list_feeds` 失败，不中断，直接改走搜索路径，并在归档时记录 `feed_failed`。

### 3. 做 2-4 轮搜索
使用 `search_feeds` 对关键词做交叉搜索，至少覆盖：
- `综合`
- `最新`
- 一个高互动维度（如 `最多点赞` 或 `最多收藏`）

分析重点：
- 是否有发布时间较近但互动异常高的黑马帖子；
- 是否出现同一类话题被重复验证；
- 哪类内容更偏点赞爆发，哪类更偏收藏/评论。

### 4. 选 3-5 条高信号候选，再取详情
对搜索结果中最有信号的 3-5 条笔记调用 `get_feed_detail`。

推荐保留字段：
- `title`
- `id`
- `xsecToken`
- `author`
- `likedCount`
- `commentCount`
- `collectedCount`
- `desc`
- 评论是否拿全 (`comments.hasMore`)

## 统一筛选口径
为了和 RedShrimp 现有 CES 思路保持一致，使用简化代理分数：

`proxy_ces = likedCount + collectedCount + commentCount * 4`

解释：
- 点赞代表广泛吸引力；
- 收藏代表“以后还要回来用”的实用价值；
- 评论乘以 4，代表讨论度和放大能力。

## 观察时必须提取的模式
每条高信号热帖，至少总结：
1. **标题钩子类型**：避雷 / 反差 / 清单 / 提问 / 情绪宣泄 / 人群定向
2. **正文结构**：先结论再展开、先情绪后细节、列表式、故事式、吐槽式
3. **情绪词**：如“避雷”“巨好吃”“谁懂”“崩溃”“值得”“救命”等
4. **评论诱导点**：二选一、让用户补充、让本地人站出来、故意留口子
5. **收藏诱因**：清单、合集、地点、模板、价格、攻略、对比
6. **场景锚点**：人群、城市、预算、约会、通勤、校园、留学生、本地生活

## 归档规则
一次实时趋势扫描成功后，必须把本次结果
追加保存到 `history/trends_data.txt`。

建议每次 run 至少包含：
- `run_id`
- `captured_at`
- `niche`
- `keywords`
- `source_status`
- `summary.top_topics`
- `summary.common_patterns`
- `summary.recommended_angles`
- `posts[]`

每条 `posts[]` 至少包含：
- `rank`
- `source`
- `topic`
- `id`
- `xsecToken`
- `author`
- `title`
- `metrics.likedCount`
- `metrics.commentCount`
- `metrics.collectedCount`
- `metrics.proxy_ces`
- `detail_desc`
- `pattern_summary`
- `capture_status`

## 输出要求
最终给用户的建议必须是具体可执行的话题，而不是泛泛地说“做美食”或“做穿搭”。

**错误示范：**
- 建议你做美食赛道。

**正确示范：**
- 最近“本地人避雷清单 + 大家来补充”的美食贴评论特别猛，你可以直接切 `广州/上海/洛杉矶本地人避雷餐厅`。
- 最近“留学生私藏店铺合集”收藏率很高，可以做 `留学生三刷以上的店` 这类可收藏型内容。
- 最近“约会 / 氛围 / 场景感”标题在城市探店里很好用，可以切 `第一次带对象来这家店值不值` 这种话题。

## Fallback：常青但稳定的黄金切口
如果 MCP 暂时不可用，可优先考虑以下方向：
1. 低门槛搞钱：副业、模板、资源、避坑
2. 打工人发疯日常：吐槽、反共识、扎心现实
3. 极端反差逆袭：前后对比、低成本升级、身份跃迁
4. 无脑听劝系列：别买什么、别去哪里、真正值得的替代方案
